方法论说明

一、为什么只需要输入 3 个数

传统统计课本里，t 检验需要输入样本均值和标准差。但在商业数据分析场景中，我们拿到的往往只有聚合指标：曝光UV、付费UV、总金额。标准差要回去翻明细、跑 SQL。

这个工具采用一个复合伯努利近似：付费行为是伯努利（付/不付），付费金额来自一个分布。基于这个模型，从"付费率 + 人均付费金额"就能反推出 ARPU 的理论方差。 这是近似方法——精度对普通决策足够，但严格论文研究请用明细数据算真实标准差。

二、AB Test 每个指标用什么方法

付费率 · Two-Proportion Z-Test
教科书经典方法，不做近似。 Z = (p̂₂ − p̂₁) / √[p̂(1−p̂)(1/n₁ + 1/n₂)] 其中 p̂ 是合并样本比例。适用于大样本（每组付费UV > 30，通常远远超过）。
ARPU · 正态近似检验
中心极限定理 + 方差分解。 Var(ARPU) ≈ Var(付费率 × ARPPU) 根据随机变量乘积的方差公式：Var(X·Y) = E[X]²Var(Y) + E[Y]²Var(X) + Var(X)Var(Y)。 付费率方差由伯努利给出，ARPPU 方差用对数正态假设近似估计（金额分布普遍偏态）。 大样本下 ARPU 近似服从正态分布，使用 Z 检验。
ARPPU · 正态近似检验
对数正态假设下的 SE 近似。 SE ≈ ARPPU × √((e^σ² − 1) / n_paid) 注意 ARPPU 分母是付费UV，样本量通常比 ARPU 小一个数量级，显著性更难达到。 默认假设对数正态 σ = 0.8（付费金额的经验值），真实业务中金额分布若更极端，SE 会被低估。

三、Ratio DiD 如何工作

本工具使用 Ratio DiD（比值差分）而非传统的减法 DiD。两者公式不同：

Ratio DiD = Tpost / Cpost − Tpre / Cpre 而标准减法 DiD 是： DiD = (Tpost − Tpre) − (Cpost − Cpre)

Ratio DiD 衡量"实验组相对对照组的位置变化"。 比如实验前 T/C = 1.00（两组水平一样），实验后 T/C = 1.20，说明实验组相对大盘高出 20%。 Ratio DiD = 0.20，带正负号反映方向。

适用场景：对照组基线和实验组不同（这很常见），但你相信对照组能作为大盘参考。 比如"老城市是实验组、新城市是对照组"，两组基线付费率差很多但都会受季节影响——此时 Ratio DiD 比减法 DiD 更稳健。

标准误用 delta method 计算： Var(T/C) ≈ Var(T)/C² + T²·Var(C)/C⁴。 Pre 和 Post 两期独立，方差直接相加。

⚠ 局限： ①当 C 很小时，比值方差爆炸，结果不稳； ②解读需要警惕——如果 T 和 C 都涨了但 T 涨得没 C 快，Ratio DiD 会变负，不代表实验"伤害"了用户； ③严格因果识别仍需多期面板+平行趋势检验，建议 R 中 fixest::feols()。

四、结果如何读

  • p 值 < α：在该显著性水平下拒绝"两组无差异"的原假设。注意 p 值不是"实验组更好的概率"。
  • 置信区间：真实效应量所在的区间。若区间跨越 0，说明不显著。
  • MDE（最小可检测效应）：当前样本量能可靠识别出的最小效应（80% power）。如果业务想看到的提升比 MDE 小，说明样本量不够，结果不显著不代表无效。
  • 相对提升：(T - C) / C，直观的业务表达。

五、什么时候不要用这个工具

样本量极小（< 100）、金额分布极度偏态（头部用户贡献 80% 营收）、存在强相关性（同一用户多次购买）、需要聚类稳健标准误——这些情况下，请用明细数据 + R 的 bootstrap 或专业统计软件。本工具适合快速决策辅助，不是论文级的严格推断。
